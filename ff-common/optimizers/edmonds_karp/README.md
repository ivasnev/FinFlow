# Алгоритм Эдмондса–Карпа

## Описание

**Алгоритм Эдмондса–Карпа** (1969) — вариант метода Форда–Фалкерсона, в котором на каждой итерации выбирается **кратчайший увеличивающий путь** (по числу рёбер) в остаточной сети. Поиск пути выполняется обходом в ширину (BFS).

В контексте оптимизации долгов:

1. Строится та же сетевая модель, что и в алгоритме Диница: источник/сток, разбиение вершин участников, рёбра по разрешённым связям и ограничениям.
2. Пока в остаточной сети есть путь от источника до стока, ищется **кратчайший** такой путь (BFS).
3. По пути проталкивается максимально возможный поток (минимум из пропускных способностей рёбер на пути).
4. Остаточные пропускные способности обновляются; шаги 2–3 повторяются.
5. Из итогового потока извлекаются переводы по разрешённым рёбрам между участниками.

Гарантированная полиномиальная сложность достигается за счёт того, что длина кратчайшего пути от источника до стока от итерации к итерации не убывает.

## Сложность

- **Время:** O(V · E²), где V — число участников, E — число долговых связей (в построенной сети рёбер порядка O(V + E)).
- **Память:** O(V + E).

Удобен как простой и предсказуемый по времени вариант; на разреженных графах часто приемлем, на плотных может уступать алгоритму Диница по скорости.
