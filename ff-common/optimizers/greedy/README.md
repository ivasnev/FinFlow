# Жадный оптимизатор долгов

## Алгоритм

Реализация метода **увеличивающих путей** (аналог Форда–Фалкерсона) для задачи упрощения долгов:

1. Граф долгов рассматривается как сеть: вершины — участники, рёбра — разрешённые переводы с пропускной способностью, равной сумме долга по этому ребру.
2. На каждой итерации ищется **произвольный путь** от любого узла с отрицательным балансом (должник) до узла с положительным (кредитор) в остаточной сети (по рёбрам с пропускной способностью > 0). Поиск выполняется BFS.
3. По найденному пути проталкивается поток величины: минимум из абсолютного баланса истока, баланса стока и минимальной пропускной способности на пути.
4. Балансы и остаточные пропускные способности обновляются; шаги 2–3 повторяются, пока существуют такие пути.

В результате получается набор переводов, сохраняющий балансы всех участников и использующий только разрешённые связи.

## Сложность

- **Время:** O(F · (V + E)), где F — суммарный поток (общий объём долгов к погашению), V — число участников, E — число долговых связей (рёбер). При целочисленных суммах число итераций не превосходит F, каждая итерация — один обход в ширину O(V + E).
- **Память:** O(V + E) для хранения графа, остаточных пропускных способностей и балансов.

Подходит для небольших и средних графов; для больших графов предпочтительны алгоритмы Диница или Эдмондса–Карпа с гарантированной полиномиальной оценкой.
